`ft classify --regex` runs a rule-based classifier over your entire bookmark corpus in under one second. It requires no LLM, no API key, and no network connection.

## Running regex classification

```bash
ft classify --regex
```

Output:

```
Classifying bookmarks (regex)...
Indexed 4751 bookmarks → ~/.ft-bookmarks/bookmarks.db

Classified 3892/4751 bookmarks (859 unclassified)

  tool            1823  (38.4%)
  technique        941  (19.8%)
  research         612  (12.9%)
  opinion          488  (10.3%)
  launch           321   (6.8%)
  security         198   (4.2%)
  commerce          87   (1.8%)
```

The command rebuilds the SQLite index and stores the classification results in the `categories` and `primary_category` columns of the `bookmarks` table.

---

## How it works

The classifier inspects three sources of signal for each bookmark:

1. **Tweet text** — the full text of the tweet is matched against pattern sets for each category.
2. **Extracted links** — URLs found in the tweet text (excluding `t.co` shortlinks) are parsed for their domain.
3. **Metadata links** — any links stored alongside the bookmark record in the JSONL cache.

Domain-based matching runs in parallel with text pattern matching. For example, a bookmark linking to `arxiv.org` is classified as `research` even if the tweet text contains no research-related keywords.

### Priority order

When a bookmark matches multiple categories, the first match in the following priority list becomes the **primary** category:

`security` → `tool` → `technique` → `launch` → `research` → `opinion` → `commerce`

All matched categories are stored (a bookmark can belong to more than one). Only the primary differs.

## Speed

The regex classifier processes your entire bookmark corpus in under one second regardless of size. It runs synchronously in the same Node.js process with no batching, no network calls, and no external dependencies.

This makes it suitable as a first pass immediately after `ft sync`, or for re-running whenever you want to update classifications without waiting for an LLM.

<Note>
  Regex classification does not assign **subject domains** (`ai`, `finance`, etc.). For domain assignment, use `ft classify` or `ft classify-domains`.
</Note>

## Rebuilding the index

`ft classify --regex` always rebuilds the SQLite index from the JSONL cache before writing classification results. This is equivalent to running `ft index` followed by classification.

If you want to rebuild the index without classification:

```bash
ft index
```

If you want to drop the index entirely and start fresh (this loses all existing classifications):

```bash
ft index --force
```

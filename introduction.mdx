Field Theory gives you self-custody over your X/Twitter bookmarks. It downloads them to your machine, indexes them for full-text search, classifies them by topic, and exposes them to any AI agent with shell access. Your data never leaves your machine.

<Note>
  Field Theory is free and open source under the MIT license. No telemetry, no analytics. Network requests go only to X's API during sync.
</Note>

## Who it's for

Field Theory is designed for people who use X/Twitter as a research and reading tool — engineers, researchers, writers, and anyone who has accumulated hundreds or thousands of bookmarks and wants to actually use them.

It works especially well as a knowledge layer for AI coding agents. You can tell Claude Code or any shell-capable agent to search your bookmarks and it will know how to use the `ft` CLI.

## How it works

```
Chrome cookies → X GraphQL API → JSONL cache → SQLite FTS5 index
                                       ↓
                              Regex / LLM classification
                                       ↓
                            Search / List / Viz / Agents
```

On first run, `ft sync` extracts your X session from Chrome's local cookie store (via macOS Keychain), uses it to authenticate a single sync request, and discards it — cookies are never written to disk separately.

Downloaded bookmarks are stored in `~/.ft-bookmarks/bookmarks.jsonl` and merged into a SQLite full-text search index. Incremental syncs only fetch bookmarks newer than your last sync.

## Data stored locally

```
~/.ft-bookmarks/
  bookmarks.jsonl         # raw bookmark cache, one record per line
  bookmarks.db            # SQLite FTS5 search index
  bookmarks-meta.json     # sync metadata
  oauth-token.json        # OAuth token (only if using API mode)
```

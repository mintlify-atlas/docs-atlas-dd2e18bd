This guide takes you from a fresh install to a working local bookmark database.

<Info>
  This quickstart uses Chrome session sync, which requires macOS and Google Chrome. If you're on Linux or Windows, see [OAuth API sync](/syncing/oauth-api) to set up `ft auth` first, then use `ft sync --api` in step 3.
</Info>

<Steps>
  <Step title="Install Field Theory">
    ```bash
    npm install -g fieldtheory
    ```

    Verify the install:

    ```bash
    ft --version
    ```
  </Step>

  <Step title="Open Chrome and go to x.com">
    Open Google Chrome, navigate to [x.com](https://x.com), and make sure you are logged in.

    Field Theory reads your Chrome session cookie to authenticate with X. No passwords are stored or transmitted — the cookie is used for the sync request and then discarded.

    <Warning>
      Chrome must be fully open (not just the window). If you recently quit Chrome, reopen it before running `ft sync`.
    </Warning>
  </Step>

  <Step title="Run ft sync">
    ```bash
    ft sync
    ```

    Field Theory extracts your session from Chrome, fetches your bookmarks from X's API, and writes them to `~/.ft-bookmarks/`. A progress indicator shows pages fetched and new bookmarks added.

    When it finishes, you'll see output like:

    ```
    ✓ 1,284 new bookmarks synced (1,284 total)
    All caught up — no new bookmarks since last sync.
    ✓ Data: /Users/you/.ft-bookmarks
    ```

    The first sync can take a few minutes depending on how many bookmarks you have. Subsequent syncs are incremental and much faster.

    <Tip>
      If you use multiple Chrome profiles, specify which one: `ft sync --chrome-profile-directory "Profile 1"`
    </Tip>
  </Step>

  <Step title="Search your bookmarks">
    Run a full-text search with BM25 ranking:

    ```bash
    ft search "machine learning"
    ```

    Results show the author handle, post date, a text excerpt, and the original URL. The search engine supports `AND`, `OR`, `NOT`, and `"exact phrases"`.

    ```bash
    ft search "rust AND async"
    ft search "\"attention is all you need\""
    ft search "kubernetes" --author karpathy --after 2024-01-01
    ```
  </Step>

  <Step title="Explore">
    Once you have bookmarks synced, these commands give you a full picture of your collection.

    Open the terminal dashboard with sparklines, category breakdowns, and top authors:

    ```bash
    ft viz
    ```

    Show how your bookmarks are distributed across categories:

    ```bash
    ft categories
    ```

    Show top authors, languages, and date range:

    ```bash
    ft stats
    ```
  </Step>
</Steps>

## What's in your data directory

After syncing, `~/.ft-bookmarks/` contains:

```
~/.ft-bookmarks/
  bookmarks.jsonl         # raw bookmark cache, one JSON record per line
  bookmarks.db            # SQLite FTS5 search index
  bookmarks-meta.json     # sync metadata (last sync time, cursor)
```

You can override the default location by setting `FT_DATA_DIR` in your environment.
